FunctorListen

FunctorListen extends FunctorTell and expands upon its ability to write to a log, by also providing a way to access the log when given a value F[A].

This means, given an F[A] we can use listen to see the accumulated value inside, by turning it into an F[(A, L)]. That comes in handy whenever you want to do something with your accumulated log. For example, we might want to send our logs to some external server. Let’s have a look at an example of how one could do this:

import cats._
// import cats._

import cats.data._
// import cats.data._

import cats.implicits._
// import cats.implicits._

import cats.mtl._
// import cats.mtl._

import cats.mtl.implicits._
// import cats.mtl.implicits._

// impure implementation for demonstrative purposes, please don't do this at home
def sendToServer[F[_]: Monad](logs: Chain[String]): F[Unit] =
  Monad[F].pure(println(show"Sending to server: $logs"))
// sendToServer: [F[_]](logs: cats.data.Chain[String])(implicit evidence$1: cats.Monad[F])F[Unit]

def sendLogsToServer[F[_]: Monad, A](logProgram: F[A])(implicit F: FunctorListen[F, Chain[String]]): F[A] =
  logProgram.listen.flatMap {
    case (a, logs) => sendToServer[F](logs).as(a)
  }
// sendLogsToServer: [F[_], A](logProgram: F[A])(implicit evidence$1: cats.Monad[F], implicit F: cats.mtl.FunctorListen[F,cats.data.Chain[String]])F[A]

Now we have a function sendLogsToServer that takes any program with logging and will send those logs to the server. To see if it works, let’s write some logging program and run it all with Writer.

// Example of some logging activity in your application
def logging[F[_]: Monad](implicit F: FunctorTell[F, Chain[String]]): F[Unit] = for {
  _ <- F.tell(Chain.one("First log"))
  _ <- F.tell(Chain.one("Second log"))
} yield ()
// logging: [F[_]](implicit evidence$1: cats.Monad[F], implicit F: cats.mtl.FunctorTell[F,cats.data.Chain[String]])F[Unit]

val result = sendLogsToServer(logging[Writer[Chain[String], ?]]).value
// Sending to server: Chain(First log, Second log)
// result: Unit = ()

And there, we can see it prints out the log. In summary FunctorListen can be used to access or “listen to” the logged values, which is pretty useful whenever you want to actually do something with the logs.
